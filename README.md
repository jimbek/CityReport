## Εισαγωγή
Στα πλαίσια της εργασίας, υλοποιήθηκε το Πληροφοριακό Σύστημα <b>CityReport</b> με τη χρήση της γλώσσας προγραμματισμού python 3.14, καθώς και με τη χρήση των βιβλιοθηκών flask και streamlit.

## Εγκατάσταση & Εκτέλεση
Η εφαρμογή μπορεί να εκτελεστεί με <b>docker desktop</b>, χωρίς να χρειάζεται να εγκαταστήσουμε κάτι επιπλέον στον Η/Υ μας. Τα βήματα που πρέπει να ακολουθήσουμε είναι:
- άνοιγμα του docker desktop και μετάβαση στο root φάκελο του project μέσω του κάτω-δεξιά terminal, εκτελώντας την εντολή <code>cd "{path-to-root-folder}"</code>
- εκτέλεση της εντολής <code>docker compose up</code> προκειμένου να δημιουργηθούν τα images και οι αντίστοιχοι containers
- επιλέγουμε τον container <b>ollama-1</b>, μεταβαίνουμε στο tab <b>Exec</b> κι εκτελούμε την εντολή <code>ollama run {model}</code> (στα πλαίσια της εργασία χρησιμοποιήθηκε το μοντέλο <code>llama3.2</code>)
για να εγκατασταθεί το μοντέλο που επιθυμούμε
- στο http://localhost:5000 εκτελείται το <b>REST API</b> και στο http://localhost:8501 το <b>Admin Dashboard</b>

## Εξωτερικό API
Χρησιμοποιήθηκε το <b>Resend</b> για την αποστολή σχολίων του Διαχειριστή με email στον εκάστοτε Πολίτη.
Καθώς χρησιμοποιήθηκε το free version, κατά την παρουσίαση της εργασίας θα δωθεί δικαίωμα αποστολής email σε αυτό που θα υποδειχθεί, προκειμένου να διαπιστωθεί η ορθή λειτουργία της υπηρεσίας.

## Βάση Δεδομένων
Χρησιμοποιήθηκε η <b>SQLite</b> για την δημιουργία της Σχεσιακής Βάσης Δεδομένων της εφαρμογής.
Ουσιαστικά, μέσω του αντίστοιχου <code>volume</code> που έχει δηλωθεί στο <code>compose.yaml</code>, όλοι οι containers διαμοιράζονται το ίδιο <code>database.db</code> αρχείο.

## Seed
Το συγκεκριμένο service είναι υπεύθυνο για την δημιουργία των πινάκων, καθώς και για την εισαγωγή κάποιων αρχικών δεδομένων.
Εκτελείται κατά προτεραιότητα, ώστε να είναι έτοιμη η Βάση Δεδομένων πριν την εκκίνηση του API και το Dashboard.

## Ollama
Το συγκεκριμένο service είναι υπεύθυνο για την εγκατάσταση κι εκτέλεση του Γλωσσικού Μοντέλου που χρειάζεται το Dashboard.
Στο http://localhost:11434/api/tags μπορείτε να δείτε τα μοντέλα που έχουν εγκατασταθεί τοπικά.

## REST API
Το συγκεκριμένο service είναι υπεύθυνο για την εξυπηρέτηση του Πολίτη. Υλοποιεί τα ακόλουθα endpoints:
- GET /states
- GET /states/{id}
- GET /categories
- GET /categories/{id}
- GET /problems
- GET /problems/{id}
- POST /problems
- PUT /problems/{id}
- DEL /problems/{id}
- GET /problems/{problem_id}/comments
- POST /problems/{problem_id}/comments

## Admin Dashboard
Το συγκεκριμένο service είναι υπεύθυνο για την διαχείριση των προβλημάτων από τους Διαχειριστές. Μέσω της διεπαφής μπορούν να:
- παρακολουθούν, εφαρμόζοντας διάφορα φίλτρα, τα προβλήματα που καταχωρούν οι πολίτες
- επεξεργάζονται το εκάστοτε πρόβλημα και να στέλνουν σχόλια (και με αποστολή email) στον πολίτη που το καταχώρησε
- επιβεβαιώνουν με τη βοήθεια του LLM για το αν η επιλεγμένη κατηγορία ενός προβλήματος είναι η πιο κατάλληλη
- βλέπουν συγκεντρωτικά στατιστικά σχετικά για τα προβλήματα ανά κατηγορία ή κατάσταση
